## Public env-var prefixes never carry secrets

<!-- pinned:guard=baseline-secret-not-public kind=baseline -->

**Past mistake:**
AI tools sometimes rename a server-only env var to start with NEXT_PUBLIC_ / VITE_ / REACT_APP_ — that inlines the secret into the client bundle.

**Rule:**
Never use a public-env-prefix (NEXT_PUBLIC_*, VITE_*, REACT_APP_*) for a variable that holds a secret (KEY, TOKEN, PASSWORD, SECRET, etc.).

**Guard:** `baseline-secret-not-public`

**Plain English:** Do not expose server secrets with public env prefixes.
## Lockfile changes require package.json changes

<!-- pinned:guard=baseline-lockfile-integrity kind=baseline -->

**Past mistake:**
AI tools sometimes regenerate the lockfile without bumping package.json — transitive deps silently shift, build becomes mystery-fragile.

**Rule:**
Do not regenerate the lockfile without a matching package.json change. If a dep update is intended, bump package.json first.

**Guard:** `baseline-lockfile-integrity`

**Plain English:** Do not regenerate the lockfile without a real dep change.
## Public package exports stay exported

<!-- pinned:guard=baseline-package-exports kind=baseline -->

**Past mistake:**
AI tools sometimes remove or rename a named export in a public entry file; consumers fail at runtime.

**Rule:**
Do not remove exported entry-point symbols without confirming no public consumer relies on them.

**Guard:** `baseline-package-exports`

**Plain English:** Do not remove exported entry-point symbols.
## CLI binaries keep working

<!-- pinned:guard=baseline-cli-exits-zero kind=baseline -->

**Past mistake:**
AI tools sometimes delete a CLI command's `bin` entry or break the `--help` invocation.

**Rule:**
Do not break the CLI binary's basic invocation (--help, --version) without checking the bin entry in package.json still resolves.

**Guard:** `baseline-cli-exits-zero`

**Plain English:** Do not break the CLI binary's --help command.
## Pinned tests and workflow are protected

<!-- pinned:guard=baseline-guard-integrity kind=baseline -->

**Past mistake:**
AI tools sometimes try to make CI green by deleting / skipping / weakening pinned tests, or by disabling the Pinned GitHub Action.

**Rule:**
Do not delete tests/pinned/* files, do not add .skip() to pinned tests, do not weaken assertions in pinned tests, do not modify .github/workflows/pinned.yml. If a pinned test is genuinely outdated, retire it: `pinned retire <claim-id>`.

**Guard:** `baseline-guard-integrity`

**Plain English:** Do not weaken pinned tests to make CI pass.
